SOAP - это протокол, по которому web-сервисы взаимодействуют друг с другом в формате xml
SOAP задает формат сообщений, которыми обмениваются клиент и сервер
представляющий собой soap envelope, который состоит из soap header (soap header необязателен), soap body и в случае ошибки soap fault 

роль - набор правил, задается атрибутом env:role (нет роли, промежуточный слой, конечный получатель)

Расширения SOAP
WS-Security - позволяющий работать с шифрованием и электронными подписями
WS-Policy

WSDL - язык описания сервисов
В WSDL-документе содержится информация о местонахождении сервиса и доступных методах (операциях); для каждого метода определяются параметры отправляемого и получаемого сообщения.

версии WSDL:
1.1 - есть разделение на типы и сообщения
2.0 - используются только типы

структура:
- описание типов
- описание интерфейсов
- биндинги
- описание местоположения сервиса

Web-сервис – это web-приложение, которое предназначено для использования другими web-приложениями

ASMX-сервис – это web-сервис, построенный в соответствии с RPC, взаимодействие происходит по протоколу SOAP

wsdl.exe - на основе wsdl генерирует proxy класс через который происходит обращение к сервису

пространства имен soap:
- soap-envelope
- soap-encoding
- soap-rpc

пространства имен wsdl:
- wsdl-instance
- wsdl-extensions
- wsdl/rpc
- wsdl/soap
- wsdl/http
